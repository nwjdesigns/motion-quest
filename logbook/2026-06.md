## 2026-06-16 — First real scene live + detail-page info panel

**Tool**: Cavalry 2.0.3 + Astro/React site  ·  **Time**: ~90 min  ·  **Tags**: pipeline, portrait, type-hierarchy, aspect-ratio

### What I made
Published the first real Cavalry scene to the live site ("Pilot: Time Offset + Stagger") and proved the publish pipeline end-to-end. Upgraded the experiment detail page to carry real metadata (tools used, Cavalry version, license) and to show the render in an aspect-correct inset container anchored to the right.

### How
- **Publish flow (run directly, no CLI):** copy `01.cv` → `public/cavalry/scenes/`, `01.png` → `public/cavalry/`, write `src/content/experiments/01.md`. Dev server 404s until content re-syncs — restart picks it up.
- **Schema (`src/schemas/experiment.ts`):** added `tools` (default `[]`), `cavalryVersion` (optional), `license` (optional), `aspectRatio` (default `16/9`). TDD: 6 new tests, 166 → 172.
- **Detail panel (`[...slug].astro`):** "Made with" section (mono-caps label + tool-tag chips), download/license + buy actions, spacing-only grouping (killed the extra dividers). Type hierarchy still flagged as weak — next task.
- **Inset container:** scene in a rounded (12px) container, `overflow: hidden`, anchored right (`justify-content: flex-end`), margins 8px top/right/bottom + 16px left, sized by per-experiment `aspectRatio` via `min(100%, calc(...))` so portrait and landscape both stay flush.
- **Homepage nodes (`ExperimentNode.tsx`):** respect each thumbnail's real aspect ratio (constant plane *area*, not hardcoded 1.6×0.9) — portrait thumbs no longer stretched.

### What clicked
The Astro content cache (`.astro/data-store.json` + `.astro/collections`) goes stale when you add a schema field AFTER content was already synced — the new field reads `undefined` in `entry.data` even though the frontmatter has it and older fields work. Clearing the cache + restarting the dev server fixes it. Lost time to this on `aspectRatio` before spotting it.
